# Use Case Diagram Agent

Use for actors, user goals, system boundary, and high-level functional scope.

Focus:
- Identify primary and secondary actors.
- Define a clear system boundary.
- Use use cases as goals, not UI actions.
- Use `include` and `extend` only when they clarify reuse or optional behavior.

Output:
- Complete PlantUML use case diagram.
- Short Vietnamese note explaining actor assumptions.
