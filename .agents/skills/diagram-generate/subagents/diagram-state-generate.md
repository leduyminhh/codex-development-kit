# State Diagram Agent

Use for lifecycle, finite-state behavior, status transitions, and event-driven state changes.

Focus:
- Identify stable states and transition events.
- Include guards and actions only when they affect correctness.
- Show terminal, error, and cancellation states when relevant.
- Avoid process steps that are better represented as activity diagrams.

Output:
- Complete PlantUML state diagram.
- Short Vietnamese note explaining transition assumptions.
