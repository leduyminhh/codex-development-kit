# Completion Handoff

Use this resource when a React task came from Notion, Slack, Linear, GitHub, or a future automation flow.

## Capture

- source ticket or thread
- requested behavior
- UI/design source
- API contract source
- files changed
- verification evidence
- known risks
- follow-up items

## Notion Payload

Keep it concise:

```text
Status:
Summary:
Files changed:
Verification:
Risks:
Follow-up:
```

## Slack Payload

Use short human updates:

```text
Done:
Checked:
Needs attention:
```

## Rules

- Do not mark external tickets complete unless the user explicitly asks.
- Do not send Slack messages unless the user explicitly asks.
- Prefer draft text when connector permissions or automation details are unclear.
